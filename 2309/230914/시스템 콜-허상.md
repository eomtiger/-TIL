# 시스템 콜
응용 프로그램의 요청에 따라 커널에 접근하기 위한 인터페이스

유저레벨의 프로그램은 유저레벨의 함수들 만으로는 많은 기능을 구현하기 힘들기 때문에, 커널의 도움을 반드시 받아야 합니다. 유저 프로세스에서 유저모드에서는 수행할 수 없습니다. 커널에 관련된 것은 커널모드로 전환한 후에야, 해당 작업을 수행할 권한이 생깁니다. 커널 모드를 통한 이러한 작업은 반드시 시스템 콜을 통해 수행하도록 설계되어 있습니다. 


#### 시스템 콜의 종류
- 프로세스 컨트롤
  - 프로세스 생성 및 종료
  - 메모리에 로드, 실행
  - 프로세스 속성 값 확인, 지정
  - 메모리 할당
- 파일 메니지먼트
  - 파일 생성, 파일 삭제
  - 열기, 닫기, 읽기, 쓰기
- 커뮤니케이션
  - 커뮤니케이션 연결 생성 및 삭제
  - 메시지 송신, 수신
- 보안
  - Permission 획득
  - Permission 설정
